# AgentForge v0.1 前端总设计文档（Usability 验证版）

## 一、文档定位
本设计文档用于指导 AgentForge v0.1 前端 Usability 验证版本的快速开发与交付。

本版本前端的本质是：
- 单智能体编辑器（Single Agent Editor）
- 后端能力展示壳（Execution Playground）
- 技术验证与立项 Demo（非商用产品）

严格定义：
- 不是工作流平台
- 不是多智能体系统
- 不是社区或产品化平台
- 不追求 UI 完整性或复杂交互

目标：**在 2 小时内完成一个可运行、可连接后端、可演示完整闭环的前端系统**。

## 二、核心设计原则

### 1. 单智能体优先
仅支持一个 Agent 的创建、编辑与执行，不设计多实例管理复杂逻辑。

### 2. 可运行优先
所有功能必须服务于“跑通闭环”，而不是视觉或完整性。

### 3. 可演示优先
必须让非技术人员能直观看到：
- 输入问题
- AI执行
- 返回结果
- 系统在“工作”

### 4. 强后端对齐
所有功能必须严格依赖当前后端能力：
- Agent 创建
- Agent 执行
- Execution Logs
- Tool / Sandbox 执行链

禁止虚构能力。

### 5. 严禁过度设计
禁止：
- 工作流画布
- 拖拽节点
- 多Agent系统
- 复杂状态管理
- 产品级交互

## 三、系统结构总览
整体采用三栏结构（非画布）：
| 左侧导航 | 中间编辑区 | 右侧实时预览 |
| --- | --- | --- |

## 四、页面结构设计

### 4.1 左侧导航栏（Sidebar）
**作用**：提供产品结构感，不承载复杂逻辑。
**模块**：
- Logo / 项目名（AgentForge）
- 控制台（占位）
- 我的智能体（当前页面）
- 知识库（占位）
- 设置（占位）

**特性**：
- 不需要真实路由
- 不需要完整功能
- 仅作为产品壳存在

### 4.2 中间编辑区（核心）
这是“单智能体构建器”，采用表单式结构。

**模块一：身份配置（Step 1）**
包含：
- 智能体名称（input）
- 简要描述（input）
- 头像（占位）
- 编辑按钮（非必须）

**模块二：指令与行为（Step 2）**
包含：
- 系统提示词（textarea）
- 开场白（input）
- 温度（slider / input）
- 最大 tokens（input）

**模块三：工具配置**
映射后端 Tool Runtime：
- 网页搜索（toggle）
- 代码执行（toggle）
- 其他工具（占位）

**模块四：知识库（弱实现）**
仅做：
- 标签展示（产品文档 / FAQ）
- 不做上传 / 管理

**中间区设计约束**：
这是表单编辑器，不是编排器。禁止节点图、流程线、拖拽逻辑、条件分支设计。

### 4.3 右侧实时预览区（核心展示区）
这是整个系统最重要的区域。

**模块一：状态栏**
显示：
- 当前状态（Ready / Running / Error）
- 工具启用状态
- 模型状态（可选）

**模块二：对话区**
结构：
- AI消息气泡
- 用户消息气泡
- 时间标记（可简化）

**模块三：输入区**
包含：
- 输入框
- 发送按钮
- loading状态

**模块四：执行反馈（关键）**
必须支持（可简化为文本提示）：
- “正在思考…”
- “调用工具…”
- “执行完成”

## 五、前后端接口映射
必须全部基于真实 API：

### 1. 创建 Agent
`POST /api/v1/agents`
- **用途**：保存当前配置、返回 `agent_id`

### 2. 执行 Agent
`POST /api/v1/executions`
- **输入**：`agent_id`、`user_input`

### 3. 查询执行结果
`GET /api/v1/executions/{execution_id}`
- **返回**：`final_result`、`execution_logs`、`react_steps`

### 4. （可选）Quota / 状态
`GET /api/v1/teams/{team_id}`

## 六、状态设计
全局状态必须极简：
- `idle`（未执行）
- `editing`（编辑中）
- `saving`（保存中）
- `running`（执行中）
- `success`（成功）
- `failed`（失败）

## 七、核心交互流程（闭环）
完整流程如下：
用户配置Agent → 点击执行 → 输入问题 → 前端调用执行接口 → 后端运行 Execution Engine → Tool / Sandbox 执行 → 返回结果 → 前端展示对话结果

## 八、演示场景（必须准备）

### 场景1：校园问答助手
- **输入**：沈航考勤规则是什么？
- **展示**：AI回答、简单总结、可展示工具调用提示

### 场景2：手册总结助手
- **输入**：帮我总结学生手册的重点
- **展示**：结构化输出、简洁总结

## 九、风险与降级策略

### 1. 模型 API 不稳定
- **处理**：显示“系统繁忙，请稍后再试”，不崩溃 UI。

### 2. Execution Logs 慢
- **处理**：先显示结果，logs 延迟加载。

### 3. 创建 Agent 失败
- **处理**：Toast 提示，保留用户输入。

### 4. 非 Bug 边界
必须明确：
- 无多智能体
- 无长期记忆
- 无复杂流程
- 无知识库管理

## 十、交付标准
必须满足：
- 页面可打开
- 能连接后端
- 能创建 Agent
- 能执行一次任务
- 能返回结果
- 能展示对话
- 不崩溃
- 可用于现场演示

## 十一、一句话总结
这是一个“单智能体执行能力展示壳”，不是一个“AI平台产品”。
