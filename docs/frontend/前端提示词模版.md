# Frontend Phase Prompt 模板（最终冻结版 v2）

---

## 模式：SOLO BUILDER

任务：Phase
为【当前 Frontend Phase】输出完整开发方案，作为唯一执行依据。

---

开始前必须阅读（强制执行，按优先级）：

1. 【最高优先级】Frontend Full Phase Plan（必须首先阅读）
   - docs/frontend/FRONTEND_FULL_PHASE_PLAN.md
   - 必须完整阅读该文档
   - 必须严格按照该文档中的 Phase 定义执行
   - 所有开发行为必须服从该文档

2. 当前 Frontend Phase 文档
   - 必须完整阅读当前 Phase 段落
   - 必须继续阅读该 Phase 中定义的“开始前必须阅读”所有文件
   - 不允许跳过 Phase 文档

3. 后端实现（必须真实读取）
   - backend/api/
   - backend/services/
   - backend/models/

   限制：
   - 仅允许读取与当前 Phase 相关模块
   - 不允许全量扫描 backend
   - 不允许推断未被使用能力

4. 后端规范文档（受控读取）
   - 仅允许读取当前 Phase 文档中列出的文件
   - 不允许扫描 docs/backend 全目录
   - 不允许读取未授权文档

---

强制规则：

1. FRONTEND_FULL_PHASE_PLAN 为唯一执行标准
2. 不允许偏离 Phase 定义
3. 不允许自行扩展 Phase
4. 不允许跳过 Phase

5. 不允许虚构 API
6. 不允许扩展 backend 能力

7. 必须识别：
   - backend 已实现能力
   - frontend 仅展示能力

8. 本项目前端限制：
   - 单 Agent
   - 无画布
   - 无多 Agent
   - 无 workflow
   - 三段式结构（左配置 / 中执行 / 右日志）

9. 工期限制：
   - 单 Phase ≤ 40 分钟
   - 总计 < 2 小时

---

输出必须包含：

- Phase目标
- Scope Freeze（必须明确不做什么）
- 页面结构（必须符合三段式）
- 状态设计
- API Mapping（必须标明来源 backend）
- Execution Path（用户操作路径）
- 风险点
- 验收标准
- 开发顺序

---

禁止：

- 设计后端功能
- 假设不存在接口
- 引入复杂组件系统
- 设计产品级 UI
- 输出解释性废话

---

## 模式：SOLO CODER

任务：
严格按照 Builder 输出实现当前 Phase。

---

开始前必须阅读（强制执行）：

1. 【最高优先级】Frontend Full Phase Plan
   - docs/frontend/FRONTEND_FULL_PHASE_PLAN.md

2. Builder 输出文档（完整）

3. 当前 Phase 文档
   - 以及其“开始前必须阅读”链路

4. backend 实现代码：
   - backend/api/
   - backend/services/

---

强制规则：

1. FRONTEND_FULL_PHASE_PLAN 为唯一执行标准
2. 不允许偏离 Phase 定义
3. 不允许扩 scope

4. 所有 API 必须来自 backend
5. 不允许 mock 后端逻辑
6. 不允许修改后端 contract
7. 不允许新增功能

---

实现要求：

1. 页面必须可运行
2. 必须完成最小闭环：
   - 创建 Agent
   - 执行 Agent
   - 展示结果
   - 展示日志

3. 优先保证：
   - 能调用接口
   - 能看到返回
   - 不追求 UI 精致

---

输出：

- CHANGED_FILES
- 使用的 API（必须列出路径）
- REAL_BACKEND_CONNECTED（YES / NO）
- STATIC_ONLY_PARTS（说明哪些是假 UI）
- RUN_METHOD
- STATUS

---

禁止：

- 自定义 API
- 使用假数据
- 绕过后端
- 输出无关说明

---

## 模式：SOLO BUILDER（AUDIT）

任务：
审计当前 Phase 前端实现是否真实成立。

---

开始前必须阅读：

1. 【最高优先级】Frontend Full Phase Plan
   - docs/frontend/FRONTEND_FULL_PHASE_PLAN.md

2. Builder 原始方案
3. Coder 实现
4. backend/api/
5. backend/services/
6. PHASE_STATUS.md

---

检查：

1. 是否严格遵循 FRONTEND_FULL_PHASE_PLAN
2. 是否真实调用 backend API
3. 是否存在假 UI（未连接后端）
4. 是否扩 scope
5. 是否违反 v0.1 限制（单 Agent / 无画布）
6. 是否形成真实闭环

---

输出：

AUDIT_SUMMARY:
- PASS / FAIL

CHECK_RESULTS:
- 是否遵循 Phase Plan：PASS / FAIL
- API真实调用：PASS / FAIL
- UI闭环：PASS / FAIL
- 是否扩 scope：YES / NO

EVIDENCE:
- 指出具体代码路径

FINAL_DECISION:
- PHASE PASSED / NOT PASSED
